Root Cause Analysis

Technical Explanation

The root cause was identified in the POST /orders/:id/process endpoint in demo-app/src/orders.ts. The payment processing logic performed arithmetic operations and division on the order.amount field without validating that it was a positive number.

Problematic Code

// Line 88-96 in demo-app/src/orders.ts
const processingFee = order.amount * 0.03; // Fails if amount is undefined
const taxRate = 0.1;
const tax = order.amount * taxRate; // Fails if amount is undefined

const total = order.amount + processingFee + tax;

// Division by zero when amount = 0
const discountEligibility = 1000 / order.amount; // 💥 CRASH!

Why It Failed

  • The POST /orders endpoint did not validate the amount field
  • Orders could be created with amount: 0 or amount: undefined
  • Payment processing attempted division by zero: 1000 / 0
  • Arithmetic operations on undefined resulted in NaN and crashes

Trigger Conditions

The error manifested when users created orders for free trials, promotional items, gift cards, or when the amount field was accidentally omitted from the request payload.

Fix Applied

Solution Description

Added validation in the POST /orders endpoint to ensure the amount field is present and greater than zero before creating an order. This prevents invalid orders from being created and eliminates the division by zero error in payment processing.

Code Changes

❌ Before (Buggy Code)
// Basic validation (but missing amount validation)
if (!customerId || !items || !Array.isArray(items)) {
  return c.json({ 
    error: 'Invalid request: customerId and items are required' 
  }, 400);
}

const order: Order = {
  id: orderId,
  customerId,
  items,
  amount, // BUG: Not validating if amount is 0 or undefined
  status: 'pending',
  createdAt: new Date().toISOString(),
};
✅ After (Fixed Code)
// Complete validation including amount
if (!customerId || !items || !Array.isArray(items)) {
  return c.json({ 
    error: 'Invalid request: customerId and items are required' 
  }, 400);
}

// NEW: Validate amount field
if (!amount || amount <= 0) {
  return c.json({ 
    error: 'Invalid amount: must be greater than 0' 
  }, 400);
}

const order: Order = {
  id: orderId,
  customerId,
  items,
  amount, // Now guaranteed to be valid
  status: 'pending',
  createdAt: new Date().toISOString(),
};
